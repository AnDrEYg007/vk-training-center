
# Журнал изменений (Changelog) - Часть 7

Этот файл содержит историю версий и описание основных изменений в приложении "VK Content Helper", начиная с версии 1.68.0.

## Версия 1.68.0 - 2024-08-07

-   **Улучшение производительности: Параллельная загрузка постов (Turbo Sync)**
    -   **Что внедрено:** Полностью переписан алгоритм синхронизации списка постов в разделе "Системные списки".
    -   **Технология:** Реализована гибридная стратегия **"Parallel Execute"**.
        1.  **VK Script (`execute`):** Вместо скачивания по 100 постов, система теперь использует скрипты, выполняемые на стороне VK, чтобы забирать пакеты по **500 постов** за один HTTP-запрос.
        2.  **Мульти-токены:** Система использует **все** доступные в базе активные токены (системные аккаунты).
        3.  **Многопоточность:** Запросы отправляются параллельно в несколько потоков, распределяя нагрузку между токенами.
    -   **Результат:** Скорость загрузки истории постов выросла в 5-10 раз. Теперь синхронизация сообщества с 2000 постов занимает секунды вместо минут, а риск временной блокировки (Rate Limit) снижен благодаря распределению нагрузки.
    -   **Стабильность:** Размер пакета `execute` был откалиброван до 500 (вместо теоретических 2500), чтобы избежать ошибки VK `Internal Server Error (Code 10)` на тяжелых данных.
