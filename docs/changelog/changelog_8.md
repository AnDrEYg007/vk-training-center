
# Журнал изменений (Changelog) - Часть 7

Этот файл содержит историю версий и описание основных изменений в приложении "VK Content Helper", начиная с версии 1.69.0.

## Версия 1.70.0 - 2024-08-09

-   **Улучшение UI/UX: Умный скролл в Системных списках**
    -   **Что внедрено:** Оптимизировано поведение интерфейса при работе с длинными списками данных (подписчики, история, посты).
    -   **Sticky Filters:** Панель с полем поиска и фильтрами теперь закрепляется (`sticky`) в верхней части экрана при прокрутке. Это позволяет менять параметры выборки в любой момент, не возвращаясь наверх страницы.
    -   **Sticky Headers:** Заголовки таблиц также закрепляются, что позволяет пользователю всегда видеть названия колонок, даже просматривая сотую строку таблицы.
    -   **Что дает пользователям:** Значительно более удобную навигацию и работу с большими объемами данных. Контекст данных (заголовки) и инструменты управления (фильтры) всегда под рукой.

## Версия 1.69.0 - 2024-08-08

-   **Оптимизация: Изменение стратегии обновления отложенных постов**
    -   **Что изменено:** Логика получения отложенных постов (`refresh_scheduled_posts`) переведена со стратегии **Broadcast** (опрос всех токенов) на **Standard Rotation** (опрос до первого успеха).
    -   **Причина:** Стратегия Broadcast необходима только для "Предложки", где видимость постов зависит от прав конкретного пользователя. Отложенные посты (postponed), как правило, либо видны админам, либо нет (Access Denied). Использование Broadcast создавало избыточную нагрузку, делая N запросов вместо 1.
    -   **Как работает сейчас:** Система берет первый активный системный токен. Если он успешно возвращает данные (даже пустой список), запрос считается выполненным. Если токен невалиден или нет доступа, система пробует следующий токен из пула.
    -   **Broadcast:** Сохранен только для **Предложенных постов**, чтобы гарантировать сбор контента, если права есть только у одного из аккаунтов.
