# Обзор архитектуры: React + Python/FastAPI

Этот документ предоставляет высокоуровневый обзор архитектуры приложения, построенного на связке React (фронтенд) и Python/FastAPI (бэкенд).

## 1. Концепция

Это классическая клиент-серверная архитектура, использующая современный и производительный стек технологий. Фронтенд отвечает за пользовательский интерфейс, а бэкенд — за всю бизнес-логику, работу с данными и взаимодействие с внешними сервисами.

## 2. Ключевые компоненты

1.  **Фронтенд (Клиент)**:
    -   **Технология**: React.
    -   **Задача**: Отображение пользовательского интерфейса, обработка действий пользователя и отправка запросов на бэкенд.
    -   **Запуск**: Работает как статическое веб-приложение в браузере пользователя.

2.  **Бэкенд (Сервер)**:
    -   **Технология**: Python с фреймворком FastAPI.
    -   **Задача**: Предоставление REST API для фронтенда, обработка бизнес-логики, взаимодействие с базой данных (SQLite) и внешними API (VK API, Gemini API), обеспечение безопасности.
    -   **Ключевая папка**: `backend_python/`

3.  **База данных**:
    -   **Технология**: SQLite, управляемая через SQLAlchemy ORM.
    -   **Задача**: Постоянное хранение всех данных приложения: проекты, кешированные посты, заметки и т.д.
    -   **Ключевой файл-схема**: `backend_python/models.py`

## 3. Схема потока данных (Типичный запрос)

```
[Пользователь] -> [Интерфейс React] -> [Вызов функции в api.ts] -> [apiClient.ts]
      |                                                                    |
      | (обновление UI)                                                    | (формирует POST-запрос)
      |                                                                    |
[Ответ JSON] <- [Бэкенд FastAPI] <- [CRUD-функции] <- [БД SQLite] / [VK API] <- [HTTP-запрос на /api/...]
```

## 4. Преимущества этого подхода

-   **Производительность**: FastAPI является одним из самых быстрых веб-фреймворков для Python, а SQLite обеспечивает быструю локальную работу с данными.
-   **Надежность**: Использование полноценной базы данных с ORM (SQLAlchemy) обеспечивает целостность данных и упрощает сложные запросы.
-   **Современный инструментарий**: Автоматическая валидация данных (Pydantic), интерактивная документация API (Swagger) и асинхронность "из коробки".
-   **Масштабируемость**: Архитектура легко масштабируется как вертикально (увеличение мощности сервера), так и горизонтально. Базу данных SQLite можно в будущем заменить на более мощную (например, PostgreSQL) с минимальными изменениями в коде.
-   **Удобство разработки**: Четкое разделение на модули (роутеры, сервисы, crud) и автоматическая перезагрузка сервера при изменениях кода.
