# 1. Основы стиля (Foundations)

Этот документ описывает базовые "строительные блоки" нашего визуального языка: цвета, шрифты и отступы.

---

## 1.1. Цветовая палитра

Мы используем ограниченную палитру для создания чистого и узнаваемого интерфейса.

### Основной (Primary)
Используется для ключевых действий, выделения активных элементов и акцентов.

- **Indigo**: `bg-indigo-600` (основной), `bg-indigo-700` (hover), `bg-indigo-100` (светлый фон).

### Нейтральные (Neutral)
Используются для фона, текста, границ и неактивных элементов.

- **White**: `bg-white` (основа для карточек и модальных окон).
- **Gray**: Широкий спектр от `bg-gray-50` (фон) до `text-gray-800` (основной текст).
  - `border-gray-200`, `border-gray-300` - для границ.
  - `text-gray-500` - для второстепенного текста.

### Семантические (Semantic)
Используются для передачи определенного смысла: успех, ошибка, предупреждение.

- **Green (Success)**: `bg-green-600`, `text-green-700`. Для кнопок "Сохранить", "Опубликовать" и индикаторов успеха.
- **Red (Danger)**: `bg-red-600`, `text-red-700`. Для операций удаления и сообщений об ошибках.
- **Amber/Orange (Warning)**: `bg-amber-100`, `text-orange-900`. Для индикации состояний, требующих внимания (например, мало постов).

---

## 1.2. Типографика

- **Основной шрифт**: `Inter`. Подключается через Google Fonts.
- **Насыщенность**: `400` (Regular), `500` (Medium), `600` (SemiBold), `700` (Bold).

### Размеры и стили:

- `text-xl font-bold`: Заголовки страниц (`<h1>`).
- `text-lg font-semibold`: Заголовки модальных окон.
- `text-base font-medium`: Заголовки секций.
- `text-sm`: Основной текст в компонентах, поля ввода.
- `text-xs`: Второстепенная информация (время, метаданные, подписи).

---

## 1.3. Отступы и сетка

- **Система отступов**: Основана на сетке `4px`. Мы используем стандартные классы Tailwind CSS (`p-4` = 16px, `gap-2` = 8px и т.д.).
- **Скругления**: `rounded-md` (6px) и `rounded-lg` (8px) для большинства элементов. `rounded-full` для кнопок-иконок и фильтров.
- **Тени**: `shadow-sm` для карточек в состоянии покоя, `shadow-md` при наведении. `shadow-xl` для модальных окон.

---

## 1.4. Анимации и переходы (Animations & Transitions)

Анимации используются для создания плавного и отзывчивого интерфейса, обеспечивая пользователю визуальную обратную связь.

### Появление элементов

-   **Эффект**: Плавное "вылетание" снизу с появлением.
-   **CSS**:
    ```css
    @keyframes fade-in-up {
        from { opacity: 0; transform: translateY(20px); }
        to { opacity: 1; transform: translateY(0); }
    }
    ```
-   **Параметры**: Длительность `0.3s`, функция `ease-out`.
-   **Применение**: Используется для появления карточек (`PostCard`, `NoteCard`), модальных окон и других блоков контента.
-   **Класс**: `animate-fade-in-up`.

### Эффекты при наведении (Hover)

-   **Эффект**: Плавное изменение цвета фона, текста или тени.
-   **CSS**: Используются утилиты Tailwind CSS.
-   **Параметры**: Длительность `200ms`.
-   **Применение**: Все интерактивные элементы (кнопки, ссылки, карточки).
-   **Классы**: `transition-colors duration-200`.

### Стабильность анимаций (Предотвращение "прыжков")

-   **Проблема**: При анимировании размеров контейнера (например, `width` или `transform` в `ProjectListItem`), дочерние элементы, чье положение зависит от этих размеров (например, при центрировании через Flexbox), могут "подпрыгивать" в конце анимации.
-   **Решение**: Для дочерних элементов внутри анимируемого контейнера следует избегать относительного позиционирования. Предпочтительный подход — **абсолютное позиционирование с выравниванием через CSS-трансформации**.
-   **Пример**: В компоненте `ProjectListItem.tsx` панель с кнопками выдвигается слева. Чтобы кнопки внутри не "прыгали", они позиционируются абсолютно и центрируются по вертикали через `top: 50%` и `transform: translateY(-50%)`. Это гарантирует, что их положение не пересчитывается в конце анимации родителя, обеспечивая идеальную плавность.
-   **Правило**: Если элемент внутри анимируемого контейнера "прыгает", попробуйте заменить `flex` или `grid` выравнивание на `position: absolute` с `transform`.

## 1.5. Форматирование данных

Единообразное отображение дат и времени важно для консистентности.

-   **Время в карточках (`PostCard`, `NoteCard`)**: Краткий формат `ЧЧ:ММ`.
    -   **Реализация**: `new Date(date).toLocaleTimeString([], { hour: '2-digit', minute: '2-digit' })`
-   **Полная дата и время (в модальных окнах, сообщениях)**: Полный локализованный формат.
    -   **Реализация**: `new Date(date).toLocaleString('ru-RU')`
-   **Дата с названием месяца (`NotePreviewModal`)**: Длинный формат для заголовков.
    -   **Реализация**: `new Date(date).toLocaleString('ru-RU', { dateStyle: 'long', timeStyle: 'short' })`

## 1.6. Адаптивность и контрольные точки (Breakpoints)

Мы используем подход **"Mobile-first"**. Стили по умолчанию предназначены для мобильных устройств, а для больших экранов добавляются префиксы Tailwind.

-   **`sm` (640px)**: Для планшетов в портретной ориентации. Сетка календаря переходит в 2 колонки.
-   **`md` (768px)**: Для планшетов в альбомной ориентации. Сетка календаря переходит в 4 колонки.
-   **`lg` (1024px)**: Для небольших ноутбуков. Сетка календаря отображается в полные 7 колонок.

**Пример из кода (`ScheduleGrid.tsx`)**:
`grid grid-cols-1 sm:grid-cols-2 md:grid-cols-4 lg:grid-cols-7`
Это означает: 1 колонка по умолчанию, 2 на `sm`, 4 на `md` и 7 на `lg` и выше.
