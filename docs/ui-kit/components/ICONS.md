# Компонент: Иконография

Этот документ является центральным каталогом всех иконок, используемых в приложении. Стандартизация иконок обеспечивает визуальную консистентность и помогает пользователям быстрее распознавать действия.

## 1. Правила использования

-   **Одна иконка — одно действие**: Не используйте разные иконки для одного и того же действия (например, только одна иконка для "Удалить").
-   **Контекст**: Иконка должна быть интуитивно понятна в контексте, в котором она используется.
-   **Доступность**: Для кнопок, состоящих только из иконки, всегда добавляйте `title` или `aria-label` для улучшения доступности.

---

## 2. Технические требования

-   **Формат**: Все иконки должны быть в формате SVG.
-   **Размеры**: Мы используем два стандартных размера, задаваемых через классы Tailwind CSS:
    -   `h-5 w-5` (20x20px): Стандартный размер для большинства иконок в интерфейсе.
    -   `h-4 w-4` (16x16px): Уменьшенный размер для компактных контекстов, таких как кнопки внутри `PostCard`.
-   **Цвет**: Все SVG-иконки должны использовать `stroke="currentColor"` или `fill="currentColor"`. Это позволяет легко управлять их цветом через классы текста (например, `text-gray-500`, `hover:text-indigo-600`).
