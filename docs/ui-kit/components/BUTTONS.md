## Компонент: Кнопки

Кнопки — основной способ взаимодействия пользователя с системой для выполнения действий.

### 1. Основные типы

#### a. Primary (Основная)
Используется для главного, позитивного действия в блоке или модальном окне (например, "Сохранить", "Запланировать", "Да").

-   **Стиль**: Заливка основным (Indigo) или семантическим (Green) цветом.
-   **Классы**: `bg-indigo-600 text-white`, `bg-green-600 text-white`.

#### b. Secondary (Вторичная)
Используется для второстепенных действий (например, "Отмена", "Закрыть").

-   **Стиль**: Светлый фон, без яркого акцента.
-   **Классы**: `bg-gray-200 text-gray-700`.

#### c. Destructive (Деструктивная)
Используется для действий, которые приводят к необратимому удалению данных (например, "Удалить").

-   **Стиль**: Красный цвет для привлечения внимания.
-   **Классы**: `bg-red-600 text-white`.

#### d. Icon-only / Text-only (Иконка / Текст)
Используется для действий внутри компонентов (например, в карточке поста) или для менее важных операций.

-   **Стиль**: Без фона, только иконка или текст.
-   **Классы**: `text-gray-500 hover:text-indigo-600`.

### 2. Состояния

Каждая кнопка имеет 4 состояния:

1.  **Default (По умолчанию)**:
    -   Описание: Стандартное состояние кнопки.
    -   Пример: `bg-indigo-600`.

2.  **Hover (Наведение)**:
    -   Описание: Состояние, когда курсор мыши находится над кнопкой. Должно быть визуально заметным.
    -   Пример: `hover:bg-indigo-700`.

3.  **Focus (Фокус)**:
    -   Описание: Состояние, когда кнопка выбрана с помощью клавиатуры (Tab).
    -   Пример: `focus:ring-2 focus:ring-offset-2 focus:ring-indigo-500`.
    -   **Исключение**: Для кнопок-переключателей (tabs, toggles) или элементов с явным изменением фона в активном состоянии, визуальная обводка фокуса может быть отключена через `focus:outline-none`, если это не нарушает доступность.

4.  **Disabled (Отключена)**:
    -   Описание: Состояние, когда действие недоступно. Кнопка неактивна и имеет соответствующий курсор.
    -   Пример: `disabled:bg-gray-400 disabled:cursor-wait`.

### 3. Правила использования

-   **Правило одного акцента**: В одной группе действий (например, в футере модального окна) должна быть только **одна** основная (Primary) кнопка. Это помогает пользователю быстро определить главное действие.
-   **Порядок кнопок**: Стандартный порядок кнопок в модальных окнах (справа налево): `[Primary] [Secondary]`. Если есть деструктивное действие, оно обычно выносится влево.
    -   **Пример**: `[Удалить] ... [Отмена] [Сохранить]`
-   **Использование Icon-only кнопок**: Этот тип кнопок предпочтителен в "тесных" интерфейсах, таких как `PostCard`, для экономии места и уменьшения визуального шума.
-   **Лаконичность и запрет переноса текста**:
    -   **Текст на кнопке должен быть максимально коротким, ясным и глагольным.** Используйте атрибут `title` для развернутых пояснений, которые появятся при наведении.
    -   **Запрещен перенос текста.** Всегда добавляйте класс `whitespace-nowrap`, чтобы кнопка не увеличивалась по высоте.
    -   **Правильно**: `<button title="Сохранить все изменения в таблице" class="whitespace-nowrap">Сохранить</button>`
    -   **Неправильно**: `<button>Сохранить все изменения</button>`
