# Архитектура Центра Обучения

**Компонент:** `features/training/components/TrainingPage.tsx`
**Данные:** `features/training/data/tocData.ts`

## 1. Источник правды (`tocData`)
Вся структура навигации определяется JSON-подобным объектом в `tocData.ts`.
*   **Иерархия:** Поддерживает вложенность (разделы -> темы -> подтемы).
*   **Path:** Каждая тема имеет уникальный `path`, который используется как ID для маппинга контента.

## 2. Динамический рендеринг (`TopicContent`)
Компонент `TopicContent.tsx` отвечает за выбор того, что показать пользователю.
*   **Mapping:** Использует объект `componentMap` для связи `path` (строка) -> `React.FC` (компонент).
*   **Lazy Loading:** Архитектура позволяет в будущем легко внедрить `React.lazy` для загрузки тяжелых интерактивных гайдов по требованию.
*   **Fallback:** Если для выбранного топика нет компонента в карте, отображается `PlaceholderPage`.

## 3. Интерактивные Демо
Обучающие материалы (например, `PostCardDeepDive.tsx`) — это не просто статический текст.
*   **Изоляция:** Демонстрационные компоненты (Mock) содержат собственное локальное состояние (`useState`), имитирующее поведение реального приложения (drag-and-drop, клики).
*   **Визуальная идентичность:** Используются специальные Mock-компоненты (например, `MockPostCard`), которые визуально копируют реальные, но не зависят от бизнес-логики и контекста приложения.

## 4. Процесс расширения
Добавление новой статьи требует трех шагов:
1.  Создать компонент контента в `features/training/components/content/`.
2.  Добавить запись в `tocData.ts`.
3.  Связать их в `TopicContent.tsx`.
