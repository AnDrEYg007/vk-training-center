# Инструкции по развертыванию

Этот документ является центральным руководством по процессу развертывания приложения "VK Content Helper" в облачной инфраструктуре Yandex.Cloud.

## 1. Общая архитектура развертывания

Приложение состоит из двух независимых частей, каждая из которых имеет свой собственный процесс развертывания:

1.  **Бэкенд (Python/FastAPI)**: Разворачивается как **Docker-контейнер** в сервисе **Yandex Cloud Containers**.
2.  **Фронтенд (React)**: Разворачивается как **статический сайт** в сервисе **Yandex Object Storage (S3)**.

## 2. Пошаговые инструкции

Для каждого компонента существует отдельная, исчерпывающая инструкция.

### ➡️ [Инструкция по развертыванию БЭКЕНДА](./DEPLOYMENT_BACKEND.md)

В этом документе вы найдете:
-   Готовый `batch`-скрипт для автоматической сборки Docker-образа и его загрузки в Yandex Container Registry.
-   Пошаговое объяснение каждой команды скрипта.
-   Инструкции по созданию новой ревизии контейнера в консоли Yandex.Cloud.

### ➡️ [Инструкция по развертыванию ФРОНТЕНДА](./DEPLOYMENT_FRONTEND.md)

В этом документе вы найдете:
-   Команды для сборки статических файлов React-приложения.
-   Готовую команду `yc storage s3 sync` для быстрой и безопасной загрузки всех файлов в бакет S3, включая автоматическую очистку старых версий.

## 3. Рабочий процесс (Workflow)

Рекомендуемый рабочий процесс включает три окружения для максимальной стабильности:

**Локальная разработка ➡️ Предпродакшен (Staging) ➡️ Продакшен (Production)**

1.  Все изменения сначала разрабатываются и тестируются локально.
2.  После завершения разработки, новая версия **сначала развертывается на предпродакшен-окружение** для финального тестирования в "боевых" условиях.
3.  **Только после успешного тестирования на предпродакшене** та же самая версия развертывается на продакшен-окружение для конечных пользователей.
