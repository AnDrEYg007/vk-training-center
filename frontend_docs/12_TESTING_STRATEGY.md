
# 10. Стратегия тестирования

Ввиду отсутствия автотестов (Jest/Cypress) на текущем этапе, качество обеспечивается строгим регламентом ручного регрессионного тестирования перед каждым релизом.

## 1. Критический путь (Critical Path)
*Эти сценарии должны работать всегда. Если они сломаны — релиз блокируется.*

### 1.1. Аутентификация
1.  Вход в систему с правильным паролем.
2.  Смена окружения API (Local -> Prod) -> перезагрузка страницы.
3.  Выход из системы.

### 1.2. Посты
1.  **Создание:** Создать пост с текстом и картинкой (загрузка с диска). Сохранить как "Системный". Проверить появление в календаре.
2.  **Drag-and-Drop:** Перетащить созданный пост на другой день. Подтвердить перемещение.
3.  **Публикация:** Нажать "Опубликовать сейчас". Убедиться, что пост появился в VK.

### 1.3. Товары
1.  Открыть вкладку "Товары".
2.  Изменить цену у любого товара. Нажать "Сохранить".
3.  Убедиться, что изменение сохранилось (перезагрузить страницу).

## 2. Смоук-тестирование (Smoke Testing)
*Быстрая проверка основных модулей после внесения изменений.*

### 2.1. Сайдбар
- [ ] Фильтры работают (поиск, команды).
- [ ] Кнопка "Обновить" на проекте запускает спиннер.
- [ ] Переход между вкладками ("Отложенные" -> "Предложенные") работает.

### 2.2. Календарь
- [ ] Сетка отображается корректно (дни недели совпадают).
- [ ] Заметки создаются и удаляются.
- [ ] Опубликованные посты (с галочкой) открываются только на чтение (дата заблокирована).

### 2.3. AI-функции
- [ ] В окне поста открывается AI-помощник.
- [ ] Генерация текста работает (нет ошибки 500).

## 3. Тестирование граничных случаев (Edge Cases)

### 3.1. Ошибки сети
1.  Отключить интернет / остановить бэкенд.
2.  Попробовать сохранить пост.
3.  **Ожидание:** Приложение не падает ("белый экран"), появляется `alert` или уведомление об ошибке сети.

### 3.2. Конфликты данных
1.  Открыть один и тот же проект в двух вкладках.
2.  В одной удалить пост.
3.  Во второй попробовать его отредактировать.
4.  **Ожидание:** Обработка ошибки 404 или корректное обновление списка.

### 3.3. Большие данные
1.  Загрузить 10 изображений в пост.
2.  Вставить очень длинный текст (5000+ знаков).
3.  **Ожидание:** Верстка не ломается, скроллы работают, данные сохраняются корректно.
