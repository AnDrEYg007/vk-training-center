
# 11. Глоссарий проекта

Единый словарь терминов, используемых в коде, документации и интерфейсе.

## Сущности

*   **Проект (Project):** Сообщество ВКонтакте, подключенное к системе. Имеет настройки, токены и контент.
*   **Пост (Post):** Единица контента. Бывает трех видов:
    *   **Опубликованный (Published):** Уже находится на стене VK.
    *   **Отложенный (VK Scheduled):** Находится в очереди `postponed` на серверах VK.
    *   **Системный (System Post):** Находится в базе данных нашего приложения (`system_posts`). Публикуется нашим планировщиком.
*   **Заметка (Note):** Внутренняя "напоминалка" в календаре. Не публикуется в VK.
*   **Предложенный пост (Suggested Post):** Пост, предложенный пользователем VK через функцию "Предложить новость".
*   **Товар (Market Item):** Товар в магазине сообщества VK.
*   **Подборка (Market Album):** Альбом (категория) для товаров в VK.
*   **Системные списки (System Lists):** Модуль для работы с локальной базой данных пользователей и активностей (подписчики, история входов/выходов, лайки, комментарии).
*   **Взаимодействие (Interaction / Activity):** Действие пользователя по отношению к контенту (лайк, комментарий, репост).

## Технические термины

*   **FSD (Feature-Sliced Design):** Архитектурная методология, используемая в проекте (разделение на `features`, `shared`, `entities`).
*   **Dirty State:** Состояние формы, когда данные в ней отличаются от исходных (пользователь что-то изменил, но не сохранил).
*   **Polling (Опрос):** Метод обновления данных, когда фронтенд периодически опрашивает бэкенд (`/getUpdates`), чтобы узнать о новых событиях.
*   **Task Monitor:** Паттерн для выполнения длительных фоновых задач (например, синхронизация тысяч подписчиков). Фронтенд запускает задачу, получает `taskId` и периодически опрашивает статус выполнения.
*   **Callback API:** Технология VK, позволяющая серверу мгновенно получать уведомления о событиях в группах.
*   **SOCKS5 Proxy:** Промежуточный сервер, используемый бэкендом для запросов к Google Gemini API для обхода региональных блокировок.
*   **Zero-Setup:** Принцип развертывания бэкенда, при котором база данных и таблицы создаются автоматически при первом запуске, не требуя ручной настройки SQL.
